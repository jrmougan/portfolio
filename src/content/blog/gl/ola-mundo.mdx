## A orixe

Cando decides crear un portfolio como desenvolvedor frontend, a tentación é obvia: facelo *bonito*. Gradientes suaves, glassmorphism, animacións fluídas, tipografías elegantes. E non está mal. Pero hai un problema: **todo o mundo fai o mesmo**.

## Por que unha terminal

A estética de terminal di algo sobre ti sen que teñas que escribilo:

- **"Sei o que hai debaixo do capó"** — Non só uso frameworks, entendo como funcionan as cousas a baixo nivel.
- **"Valoro a funcionalidade"** — O contido importa máis que o envoltorio.
- **"Teño criterio propio"** — Non sigo tendencias por inercia.

## A decisión técnica

Escollín [Astro](https://astro.build) porque xera HTML estático puro. Sen runtime de JavaScript innecesario. Un portfolio non necesita hidratación do lado do cliente para mostrar texto.

Para as fontes, **Geist Pixel Square** de Vercel para os headings e **Geist Mono** para o corpo. Todo self-hosted, sen dependencias de Google Fonts.

A paleta está inspirada en [Dracula Theme](https://draculatheme.com/), que é posiblemente o tema de terminal máis recoñecible do ecosistema dev.
